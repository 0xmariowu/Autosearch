## Content types

Code repositories on GitHub, including libraries, frameworks, apps, SDKs, templates, and example implementations. Results usually include repo name, description, stars, forks, primary language, and update metadata.

## Language

Works for any programming language because code itself is the main artifact. Natural-language metadata is still often English, so obscure non-English query phrasing may match less well than code terms or package names.

## Best for

Finding tools, libraries, frameworks, starter kits, and real implementations of a feature or technique. This is the strongest channel when the answer should be "show me the repo."

## Blind spots

It does not find articles, tutorials, papers, release notes, or broad discussions unless they happen to live inside a repository. It also says very little about whether a repo is easy to use, well maintained, or correct beyond visible repo metadata.

## Quality signals

Star count is the strongest quick signal, especially for mature tools. Recent activity matters, and a strong description, sensible language match, and active maintenance are better signals than name similarity alone.
