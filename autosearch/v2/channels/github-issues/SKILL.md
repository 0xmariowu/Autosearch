## When to use

Use when the best evidence is in GitHub issues, bug reports, or request threads rather than repository listings.

## Quality signals

- Issues from relevant repositories
- Clear problem statements and reproducible symptoms
- Recent creation dates for active problems

## Known limits

- Requires the `gh` CLI to be installed and authenticated
- Body snippets are truncated
- Search may include noisy issue titles with loose keyword overlap
